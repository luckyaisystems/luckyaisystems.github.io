# LuckyAISystems — Repo Families & Artifact Types (v1.0)

This document defines the major repository families inside LuckyAISystems and the types of artifacts each family should contain.  
Its purpose is to keep the ecosystem modular, scalable, and easy for both humans and AI agents to navigate.

---

# 2. Artifact Types

Every repo should contain four types of artifacts.

## 1. Source-of-Truth Docs  
These define:
- principles  
- standards  
- workflows  
- architecture  

Examples:
- Communication OS  
- Operation Capability  
- NEXT_DECISION  

---

## 2. Generated Artifacts  
These are produced by:
- Co-pilot  
- AI workflows  
- templates  
- compilers  

Examples:
- structured SOPs  
- JSON schemas  
- documentation pages  

---

## 3. Experimental Ideas  
These are drafts, prototypes, or early concepts.

Examples:
- new module ideas  
- new repo concepts  
- early workflows  

These should **not** be treated as permanent artifacts.

---

## 4. Decision Tools  
These help you make better choices.

Examples:
- mortgage readiness engine  
- housing transition planner  
- promotion tracker  
- expense leak analyzer  

These are **not** documentation — they are calculators and evaluators.

---

# 3. High-Value Missing Repos

These are the strongest opportunities Perplexity identified.

## A. SOP Generator  
Turns a situation into a structured SOP template.

## B. README / Docs Generator  
Converts raw notes into clean documentation pages.

## C. Promotion Tracker  
Models salary jumps, experience milestones, and role progression.

## D. Expense Leak Analyzer  
Detects lifestyle creep and quantifies savings loss.

## E. Equinix Site Notes Compiler  
Stores observations → converts them into patterns → links them to portfolio artifacts.

---

# 4. Agent Control Layer

Each major repo should contain an **instruction file** that tells AI agents:

- purpose  
- allowed operations  
- structure  
- evaluation rules  
- guardrails  

This makes LuckyAISystems compatible with modern agentic markdown patterns.

---

# 5. Extraction Rule

An idea belongs in LuckyAISystems if it satisfies at least two:

- improves communication  
- improves critical facilities capability  
- improves financial/housing decisions  
- becomes a reusable system  
- supports long-term career progression  

This rule prevents unnecessary repo creation and keeps the ecosystem lean.
